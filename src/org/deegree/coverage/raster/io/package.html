Defines the io operations for reading and writing rasters (not the data).

An implementation of data RasterIOProvider provides some Implementations of the RasterReader RasterWriter interface. The loading of such a provider is loaded using the ServiceLoading mechanism, the default deegree 3 io providers are contained in the d3_dataaccess module. Current implementations are (see d3_dataaccess/src/META-INF/):

  • org.deegree.dataaccess.imageio.IIORasterIOProvider
  • org.deegree.dataaccess.jai.JAIRasterIOProvider

Related Documentation

A RasterIOProvider provide a RasterReader, which provides methods about the possible file types it can handle, but the loading of the actual data (the raster/coverage images) is done by supplying an implementation of a RasterDataReader, which provides methods to retrieve a datacontainer.

Both above mentioned implementations supply implementations of these RasterDataReaders. For example:

the org.deegree.dataaccess.imageio.IIORasterIOProvider provide the org.deegree.dataaccess.imageio.IIORasterReader (an implementation of the org.deegree.commons.model.raster.io.RasterReader) which adds the possible file types of imageIO. It then uses org.deegree.dataaccess.imageio.IIORasterDataReader (an implementation of org.deegree.commons.model.raster.io.RasterReader) to access the images/raster files.
